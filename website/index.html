Built for the library

A few reasons to keep it in your toolkit.

01

Batch conversion

Convert a file or point flacser at a directory. Recursive discovery is there when your library runs deep.

02

Parallel by default

Independent conversions run in parallel, with a worker limit you can set when you need to.

03

Structure stays put

Recursive directory conversions preserve paths from your input root instead of making a mess of your collection.

04

Metadata, best effort

flacser lets ffmpeg carry metadata where the output container supports it.

No surprises

Careful with the files you care about.

Before conversion starts, flacser plans output paths and detects collisions. Outputs already present when their job starts are skipped.

Each conversion writes to a temporary file first, then moves it into place only when ffmpeg succeeds. Source files are never modified.

Get started

Install flacser your way.

flacser requires ffmpeg and ffprobe on your PATH.

Install with Cargo

cargo install flacser

Install the latest published flacser release.

Build from source

cargo build --release

Run the resulting binary at target/release/flacser.

Usage

Your next conversion, sorted.

Convert a directory

flacser convert ./music --to aiff --recursive --output-dir ./converted

Use --dry-run to run conversion planning without running ffmpeg or ffprobe.

Check your setup

flacser doctor ./music --to aiff

Doctor is read-only and reports whether your system is ready.
